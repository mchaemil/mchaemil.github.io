### 데이터 타입

메모리상에서 똑같은 값은 메모리 하나만을 차지한다. 
그래서 주소낭비를 하지 않는다. 

> 참조카운트

```javascript
var a;
a = "a";
a = "abcde";
```

값으로 주소를 가지고 있으므로 기본형에 비해 한 단계 더 나아가서 차이가 없다

```javascript
var object = {
  a: 1,
  b: 'text',
}

object.a = 2;
```

참조하고 있는 값이 없으면 가비지 카운팅을 당한다.
참조카운트가 0이 되면, 즉 값을 참조하고 있는 대상이 몇개 인지 세어서

```javascript
var obj = {
  a: 1,
  arr: [3, 4],
}
obj.arr = 'str';
```

#### JS의 숫자!
자바스크립트는 태생적으로 정확한 수 계산을 위해서 만들어진 언어가 아니므로 정밀하고, 범용성이 높은 규격을 선택했다!(숫자에 대해) 

자바스크립트는 IEEE 754(아이트리플E 754)라는 규격을 따르고 있다
IEEE 754는 크기가 64비트이기 때문에 정밀한 값을 표현하는데 좋다.
플로팅 포인트 넘버(floating point number), 부동소수점 숫자 = 정밀한 숫자!!   
정밀하다는 것은 정확하다는 뜻이 아니다. 
1.0 과 1은 다르다. 1.0은 1에 아주 근사(근접)한 수이다. 


#### 포인트 넘버(fixed point number)
픽스드 포인트 넘버(fixed point number) 즉, 고정소수점 숫자
정수는 항상 오른쪽에 소수점이 고정되어 있어서 
소수부를 사용하지 않는다. 

#### 정확한 수는 계산할 때 사용한다!!

스탑워치는 정밀해야 한다. 
달리기 시합은 1000분의 1초까지 정밀하게 계산하지만 이 범위를 벗어난다면 오차를 허용한다. 

은행에서 돈의 계산은 정확해야 한다. 1원이라도 차이가 생기면 곤란하다.
위의 예시를 통해서 정밀과 정확의 본질에 대해 이해할 수 있다.
